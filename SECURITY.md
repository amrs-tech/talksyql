# Security Policy

talksyql is an experimental learning project. Security reports are still welcome, but response times are best-effort.

Please report vulnerabilities privately by emailing:

```text
amrs.tech@gmail.com
```

Do not post secrets, API keys, database URLs, authentication bypasses, exploit details, or sensitive user data in public GitHub issues or discussions.

When reporting, include:

- A clear description of the issue.
- Steps to reproduce, if safe to share privately.
- The affected version, commit, or deployment context.
- Any recommended fix or mitigation, if known.

This project does not currently offer a formal bug bounty program.
